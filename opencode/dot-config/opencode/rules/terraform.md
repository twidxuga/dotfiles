# Terraform Rules

## Style
- Use 2-space indentation
- Align `=` signs in attribute blocks
- Order: `terraform` → `required_providers` → `provider` → `data` → `resource` → `output`
- Use `_` not `-` in resource names

## Modules
- Always use versioned module sources (never `?ref=main`)
- Document all variables with `description` and `type`
- Provide `default` values only for optional variables

## State
- Never manually edit state files
- Always use remote state (S3 + DynamoDB or Terraform Cloud)
- Use `terraform state` commands for state manipulation

## Testing
- Write `.tftest.hcl` tests for all new modules
- Mock external providers in unit tests
- Run `terraform test` before marking module work complete

## AWS Service Control Policies (SCPs)
- Before creating any new S3, IAM, or other AWS resources, check `policies/scps/*.json.tpl` for relevant restrictions — org SCPs can deny actions that would otherwise succeed
- `aws_s3_bucket_public_access_block` is **blocked by the SCP `DenyPublicS3Buckets`** for all workload roles (including `GitHubActionsDeployRole`) — omit it and add a comment referencing the SCP (see `modules/email-service/main.tf` for the established pattern)
- The account-level public access block (enforced by the security baseline) already covers all buckets — no bucket-level resource is needed or allowed

## IRSA Trust Policies
- Always check `live/dev/main.tf` and `live/dev/hellobill.tf` for existing IRSA conventions before writing a new trust policy
- Kubernetes namespace in trust conditions must be `bunch-platform` (not `bunch-apps`) — all services run in `bunch-platform`
- Service account names follow the Helm `fullname` prefix: `bunch-apps-{service}-sa` (generated by `_helpers.tpl` as `printf "%s-{service}-sa" (include "bunch-apps.fullname" .)`)
- The dynamic env trust condition prefix is `preview-*` (not `preview-bun-*`) — changed in PR #360
- Verify Bedrock model ARN versions against actual app code defaults (e.g. `bedrock.service.ts`) before hardcoding in IAM policies — version mismatch causes `AccessDeniedException`
